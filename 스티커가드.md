# 📱 "스티커 가드" 앱 기획서

## 📋 문서 정보

**작성일**: 2025.11.02  
**버전**: 1.0  
**작성자**: Product Planning Team  
**문서 목적**: 회사 보안 정책 준수를 위한 카메라 스티커 관리 앱 기획

---

## 🎯 제품 개요

### 1.1 배경 및 목적

**문제 인식**

- 많은 기업들이 보안을 위해 직원들에게 카메라 스티커 부착을 요구
- 하지만 수동 관리로 인한 실수와 누락이 빈번히 발생
- 보안 위반 시 조직과 개인 모두에게 큰 리스크 발생

**해결 방안**
자동화된 시스템을 통해 사람의 실수를 원천 차단하고, 100% 준수율을 보장하는 앱 개발

### 1.2 제품 컨셉

```
"설치하면 끝. 보안 위반 0% 보증"

✓ 회원가입 없이 바로 시작
✓ 회사 위치만 설정하면 자동으로 작동
✓ 카메라로 스티커 직접 확인하여 확실성 보장
✓ 강제 체크인 시스템으로 실수 방지
```

### 1.3 핵심 가치 제안

**기존 방식의 문제점**

- ❌ 수동으로 기억해야 함 → 까먹기 쉬움
- ❌ 자가 검증만 가능 → 확신 부족
- ❌ 강제성 없음 → 나중에 하다가 누락
- ❌ 복잡한 가입 절차 → 시작하기 어려움

**스티커 가드의 해결책**

- ✅ 회원가입 없음 → 설치 후 바로 시작
- ✅ 자동 알림 → 놓칠 수 없음
- ✅ 카메라 확인 → 육안으로 확실하게
- ✅ 강제 체크인 → 위반 불가능

### 1.4 타겟 사용자

**Primary Target**

- 보안이 중요한 기업의 직원 (금융, IT, 제조업 등)
- 카메라 스티커 부착이 의무인 조직
- 출퇴근이 규칙적인 직장인

**Secondary Target**

- 보안 관리 담당자 (컴플라이언스 확인 용도)
- 재택근무와 사무실 근무를 병행하는 하이브리드 워커

## 🔑 핵심 기능 명세

### 2.1 자동 위치 감지

**기능 개요**
회사 위치에서 이탈했다가 다시 진입하면 자동으로 푸시 알림을 보내 체크인을 유도

**상세 스펙**

- **감지 범위**: 회사 반경 300m (조정 불가)
- **작동 방식**: 백그라운드에서 24시간 GPS 추적
- **트리거 조건**:
  - 회사 반경에서 완전히 이탈 (300m 이상 멀어짐)
  - 이후 다시 회사 반경 내 진입
  - 당일 미체크 상태일 경우에만 알림
- **배터리 최적화**: 하루 5% 미만 소모 목표

**사용자 경험**

1. 사용자가 집에서 출발 (회사 반경 밖)
2. 회사 근처에 도착 (300m 이내 진입)
3. 자동으로 "출근 체크 필요" 푸시 알림 수신
4. 알림 클릭 시 바로 체크인 화면으로 이동

**예외 처리**

- 이미 당일 체크 완료 시 알림 없음
- 회사 안에서 계속 머물러 있는 경우 알림 없음 (이탈 후 재진입이 트리거)
- 위치 권한 거부 시 앱 사용 불가 안내

### 2.2 카메라 차단 검증

**기능 개요**
사용자가 직접 후면 카메라를 켜서 스티커 부착 여부를 육안으로 확인하는 방식

**검증 원리**

1. 사용자가 "카메라 확인" 버튼 클릭
2. 후면 카메라 화면이 전체 화면으로 표시됨
3. 사용자가 카메라를 통해 스티커 부착 여부를 직접 확인
4. 확인 후 "스티커 있음" 버튼 클릭

**iOS 정책 준수**

- iOS는 백그라운드 카메라 자동 실행을 허용하지 않음
- 사용자가 명시적으로 버튼을 클릭해야 카메라 활성화
- 사용자가 화면을 보고 직접 판단하는 구조

**사용자 가이드**

- 폰을 45도 각도로 들어 후면 카메라가 보이게 하기
- 카메라 화면에서 스티커가 부착되어 있는지 확인
- 스티커가 있으면 "확인 완료" 버튼 클릭

**처리 시간**

- 사용자 확인: 5-10초
- 버튼 클릭 후 즉시 완료

### 2.3 강제성 시스템

**핵심 철학**
"회사 근처에 왔으면 반드시 체크해야 한다"는 강력한 강제 메커니즘

**규칙**

- 하루 1회 필수 체크 (회사 도착 시)
- 회사 300m 반경 진입 후:
  - 45분 이내 체크 필수
  - 45분 초과 시 계정 즉시 잠금
- 계정 잠금 시:
  - 모든 데이터 삭제 (연속 기록, 통계 등)
  - 앱 사용 불가
  - 앱 재설치 필요
  - 처음부터 다시 설정

**알림 강화**
회사 300m 진입 후:

- 즉시 (0분) - "💚 입문 전 스티커 체크 필수!"
- 5분 후 - "⚠️ 아직 체크 안 함 - 지금 바로 체크하세요"
- 15분 후 - "🚨 체크하지 않으면 앱 사용 불가!"
- 30분 후 - "❗❗ 마지막 경고 - 곧 계정 잠금"
- **45분 후 미체크 시 계정 즉시 잠금**

**페널티 메커니즘**

- 연속 기록 손실 (예: 48일 → 0일)
- 획득한 모든 배지 삭제
- 통계 데이터 완전 삭제
- 복구 불가능 (백업 없음)

**설계 의도**

- 회사 입문 전 반드시 체크하도록 강제
- 45분이라는 충분한 시간 제공 (입문 전 여유)
- 체크 안 하면 즉시 패널티 (복구 불가능한 데이터 손실)
- 자발적 설치 사용자의 의지를 앱이 강제 실행으로 보조
- 강력한 습관 형성 촉진

### 2.4 통계 및 연속 기록

**연속 기록 (Streak)**

- 연속으로 체크한 일수 표시
- 불 이모지(🔥)로 시각화
- 홈 화면 중앙에 크게 표시

**통계 정보**

- 총 체크인 횟수
- 완벽한 주 (주 5일 모두 체크)
- 최장 연속 기록
- 월별 달성률

**배지 시스템**

- 7일 연속 - "일주일 마스터"
- 30일 연속 - "한 달 챔피언"
- 100일 연속 - "백일 전설"
- 365일 연속 - "완벽주의자"

## 👤 사용자 플로우

### 3.1 최초 사용 플로우

**Step 1: 다운로드 및 설치**

```
앱스토어 다운로드
→ 설치
→ 앱 실행
→ 환영 화면
→ "시작하기" 버튼
```

**Step 2: 회사 위치 설정 (30초)**

```
"회사 위치를 알려주세요"
→ 방법 선택:
   A. "지금 여기가 회사입니다" (현재 위치 사용)
   B. "주소 검색" (수동 입력)
→ 지도에서 확인
→ "확인" 버튼
```

**Step 3: 권한 설정 (1분)**

```
"필수 권한을 허용해주세요"
→ 위치 (항상 허용) ← 가장 중요
→ 카메라
→ 알림
→ 백그라운드 실행
→ 각 권한마다 OS 설정 팝업
```

**Step 4: 테스트 체크인 (30초)**

```
"첫 체크인을 해볼까요?"
→ "테스트 시작" 버튼
→ 카메라 검증 실행
→ 결과 확인
→ "완료" 버튼
→ 홈 화면으로 이동
```

**총 소요 시간: 약 2-3분**

### 3.2 일상적인 사용 플로우

**아침 (출근 시)**

```
집에서 출발 (회사 반경 밖)
→ 회사 근처 도착 (300m 이내 진입)
→ 자동으로 푸시 알림 수신
→ 알림 클릭
→ 앱 실행
→ "카메라 확인" 버튼
→ 후면 카메라 화면 표시
→ 스티커 부착 확인 (육안)
→ "확인 완료" 버튼 클릭
→ ✅ "체크 완료! 48일 연속"
→ 앱 종료
```

**체크를 놓쳤을 때**

```
회사 도착 (300m 이내)
→ 즉시 알림 무시
→ 5분 후 - "아직 체크 안 함" 알림 무시
→ 15분 후 - "앱 사용 불가 경고" 알림 무시
→ 30분 후 - "마지막 경고" 알림 무시
→ 45분 경과
→ ⚠️ 계정 즉시 잠금
→ 모든 데이터 삭제
→ 앱 재설치 필요
```

**이미 체크한 날**

```
출근 체크 완료
→ 회사 내에서 하루 종일 근무
→ 퇴근 (회사 이탈)
→ 알림 없음 (이미 체크 완료)
→ 다음날까지 앱 사용 불필요
```

### 3.3 계정 잠금 플로우

**시나리오: 회사 도착 후 45분 내 체크 못 함**

```
회사 반경 진입 (예: 오전 9시)
→ 즉시 알림 받음 → 무시
→ 5분 후 (9:05) 알림 → 무시
→ 15분 후 (9:15) 경고 알림 → 무시
→ 30분 후 (9:30) 마지막 경고 → 무시
→ 45분 경과 (9:45)
→ 시스템이 자동으로 감지
→ 모든 데이터 삭제 실행
→ 계정 상태를 'LOCKED'로 변경
→ 앱 즉시 사용 불가

앱 실행 시도 시
→ 잠금 화면 표시
→ "🔒 계정이 잠겼습니다"
→ 잠금 사유: "45분 내 체크 미완료"
→ 잃어버린 것 표시:
   - 48일 연속 기록
   - 획득한 모든 배지
   - 통계 데이터
→ 재시작 방법 안내:
   1. 앱 삭제
   2. 재설치
   3. 처음부터 설정
→ "앱 설정으로 이동" 버튼
```

## ⚠️ 리스크 및 대응 방안

### 7.1 기술적 리스크

**Risk 1: 위치 추적 배터리 소모**

- 문제: 백그라운드 GPS 사용으로 배터리 과다 소모
- 영향도: 높음
- 대응:
  - 위치 업데이트 주기 최적화 (15분 간격)
  - Geofencing API 활용 (진입/퇴장만 감지)
  - 저전력 모드 지원
  - 사용자에게 배터리 최적화 가이드 제공

**Risk 2: 카메라 검증 오판**

- 문제: 조명 조건에 따른 오판 가능성
- 영향도: 중간
- 대응:
  - 다양한 환경에서 충분한 테스트
  - 재시도 기회 제공 (3회)
  - 사용자 피드백 수집 및 알고리즘 개선
  - 수동 리뷰 시스템 (고객 지원)

**Risk 3: OS 업데이트 호환성**

- 문제: iOS/Android 업데이트로 기능 동작 불가
- 영향도: 중간
- 대응:
  - 베타 OS 버전에서 사전 테스트
  - 최소 OS 버전 요구사항 명시
  - 빠른 업데이트 대응 체계

### 7.2 사용자 경험 리스크

**Risk 4: 강제성에 대한 반발**

- 문제: 지나치게 강한 패널티로 사용자 이탈
- 영향도: 높음
- 대응:
  - 명확한 사전 고지 (온보딩 시)
  - 점진적 알림으로 충분한 기회 제공
  - 정당한 사유 (출장, 병가) 예외 처리 검토
  - 사용자 피드백 지속 수렴

**Risk 5: 알림 피로감**

- 문제: 잦은 알림으로 사용자 거부감
- 영향도: 중간
- 대응:
  - 알림 설정 커스터마이징 옵션
  - 시간대별 점진적 알림 (처음엔 부드럽게)
  - "방해 금지" 시간대 설정

### 7.3 운영 리스크

**Risk 6: Firebase 비용 증가**

- 문제: 사용자 증가로 인한 서버 비용 급증
- 영향도: 중간
- 대응:
  - 초기에는 Firebase 무료 플랜으로 충분
  - 사용자 1,000명 이상 시 유료 플랜 전환 예산 확보
  - 비용 모니터링 및 최적화
  - 필요시 다른 백엔드로 마이그레이션 검토

**Risk 7: 개인정보 보호 문제**

- 문제: 위치 정보 수집에 대한 개인정보 우려
- 영향도: 중간 (익명 인증으로 완화)
- 대응:
  - 익명 인증 사용 (개인 식별 정보 불필요)
  - 명확한 개인정보처리방침 작성
  - 최소한의 데이터만 수집 (위도/경도만)
  - 서버에 위치 이력 저장 안 함
  - 암호화 전송 (HTTPS)
  - 정기적인 보안 감사

### 7.4 규제 리스크

**Risk 8: 앱스토어 정책 위반**

- 문제: 강제 재설치 메커니즘이 정책 위반 가능성
- 영향도: 높음
- 대응:
  - 사전에 앱스토어 가이드라인 철저히 검토
  - 필요시 법률 자문
  - 명확한 사용자 동의 절차
  - "계정 삭제" 옵션 제공 (강제가 아닌 선택)
